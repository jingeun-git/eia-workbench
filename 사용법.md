# EIA Workbench 사용법

> 웹 UI: **https://jingeun-git.github.io/eia-workbench/** — 설치 없이 접속만 하면 됩니다.

## 0. 두 부분으로 되어 있습니다

이 도구는 **웹**과 **브리지** 두 프로그램으로 나뉩니다. 이걸 알아야 아래 내용이 이해됩니다.

| | 웹 | 브리지 |
|---|---|---|
| 무엇 | 지금 보는 화면. 그냥 인터넷 주소 | PC에서 도는 작은 프로그램 |
| 실행 | **실행할 것이 없습니다.** 접속하면 끝 | `run_bridge.bat` 더블클릭 |
| 갱신 | 새로고침하면 항상 최신 | 다시 실행해야 반영 |
| 버전 표시 | 왼쪽 위 `웹 vX.Y.Z` 배지 | 오른쪽 위 상태칩 `● 브리지 vX.Y.Z` |

> **두 버전 숫자가 다른 것이 정상입니다.** 별개 프로그램이라 각자 필요할 때만 올라갑니다.
> 브리지는 PC에서 하는 일(한컴·파일 접근)이 바뀔 때만 갱신되므로 웹보다 낮은 것이 보통입니다.
> 실제로 맞지 않으면 상태칩에 **⚠ 갱신 필요**가 뜹니다 — 그때만 브리지를 다시 실행하세요.

## 1. 도구 한눈에 보기

탭은 **업무 흐름 3그룹**으로 묶여 있습니다.

### 자료 수집·조사

| 도구 | 브리지 | 하는 일 |
|---|---|---|
| EIASS 수집 | ❌ (사업코드 자동탐색만 ⭕) | 협의 원문 PDF 다운로드 |
| 문서 → MD | ❌ (HWP·스캔 OCR만 ⭕) | PDF·DOCX·XLSX → 마크다운 |
| 사진 좌표 | ⭕ | 현장사진 EXIF → 촬영지점·방향 지도 표시 → CSV·KML |
| 지오코딩 | ❌ | 주소↔좌표 양방향 · 지도 더블클릭 조회 → 엑셀 |

### 본문 작성

| 도구 | 브리지 | 하는 일 |
|---|---|---|
| 건축물대장 | ⭕ | 사업지구 SHP/주소 → 필지별 건축물대장 → Excel |
| PDF 표 → 엑셀 | ⭕ | PDF 안의 표만 골라 엑셀로 (연속 표 자동 병합) |

### 제출본 정리

| 도구 | 브리지 | 하는 일 |
|---|---|---|
| 쪽번호 | ⭕ | 장별 쪽번호 일괄 부여 (간지·A3 양면 정합) |
| HWP → PDF | ⭕ | 폴더 단위 한컴 일괄 변환 |

**브리지가 꺼져 있어도 ❌ 도구는 전부 정상 동작합니다.**

## 2. 최초 1회 — API 키 설정

1. 웹 UI 접속 → 우상단 **⚙ 설정**
2. **vworld API 키**: [vworld.kr](https://www.vworld.kr) 회원가입 → 인증키 발급
   — 건축물대장·지오코딩·지도 배경(위성영상)에 쓰입니다
3. **공공데이터포털 키**: [data.go.kr](https://www.data.go.kr) → "건축HUB 건축물대장 정보 서비스" 활용신청
   → 마이페이지의 **일반 인증키(Decoding)** 입력
4. 저장 — 키는 **이 브라우저의 localStorage에만** 남고 어디에도 전송되지 않습니다

> vworld 키가 없으면 지도 배경이 OpenStreetMap으로 대체되고 위성영상을 볼 수 없습니다.

## 3. 브리지 실행 (⭕ 도구를 쓸 때만)

**`D:\Claude\99.Tools\unified_workbench\bridge\run_bridge.bat` 더블클릭 — 끝.**

브라우저가 자동으로 열리면서 토큰이 스스로 등록되고, 상태칩이 **● 브리지 vX.Y.Z**로 바뀝니다.

- 요구사항: Windows Python **3.10+** (한컴 작업은 한컴오피스 + pywin32 추가)
- **콘솔 창 없이 트레이에 상주**합니다 — **웹 UI를 닫으면 곧 자동 종료**됩니다(약 10초).
  탭을 여러 개 열어두었다면 마지막 탭을 닫을 때 종료됩니다.
- **이미 실행 중이면 새로 뜨지 않습니다** — exe를 다시 눌러도 웹 UI만 열립니다(v3.25.0).
- 자동으로 안 열리면: 콘솔에 표시된 토큰을 웹 UI ⚙ 설정에 수동 입력
- 토큰은 `bridge/bridge_config.json`에 저장 — **이 파일은 공유·커밋 금지**

## 4. 도구별 사용법

### 사진 좌표 (브리지)
- [폴더 선택] → [사진 읽기]. 폴더의 사진에서 촬영지점·방위각·화각을 읽습니다.
- 지도에 촬영지점이 찍히고, **활성 사진만 화각 삼각형과 방향선**이 그려집니다.
- **양방향 선택**: 목록에서 고르면 지도가 이동하고, 지도 표시를 누르면 목록이 선택됩니다.
- 내보내기는 **CSV**(엑셀·QGIS) 또는 **KML**(구글어스). 사진 파일명이 속성으로 함께 저장됩니다.
- **좌표가 없는 사진도 목록에 남습니다** — 사유가 표시되므로 몇 장이 빠졌는지 알 수 있습니다.
- ⚠ 아이폰 HEIC는 `pillow-heif`가 설치돼 있을 때만 읽힙니다(대부분 JPG로 저장되므로 해당 없음).

### 지오코딩 (브리지 불필요)
- **파일로**: [파일 선택] → **어느 열을 쓸지 고르는 화면**이 먼저 뜹니다.
  열 이름으로 후보를 제안하지만 **확인은 직접** 하세요 — 코드·관리번호처럼 숫자로 된 열이
  좌표로 오인될 수 있습니다. 앞 3행 미리보기를 함께 보여줍니다.
- **입력 좌표계는 자동 추정하지 않습니다.** 원점을 잘못 고르면 오류 없이 수백 km 어긋납니다.
  값의 생김새가 좌표계와 맞지 않으면 **권장 좌표계를 알려주고 한 번에 바꾸는 버튼**이 나옵니다.
- **지도로**: 지도를 **더블클릭**하면 그 자리의 주소를 조회해 목록에 추가합니다.
  이름은 `지점-1`부터 자동 부여되고 목록에서 바로 고칠 수 있습니다.
- 두 방법으로 모은 지점은 **한 목록**에 쌓이고, 내보낸 엑셀에서 **출처** 칼럼으로 구분됩니다.
- 체크한 행이 있으면 **조회·내보내기 모두 그것만** 처리합니다(없으면 전체).
- [열 매핑 수정]으로 열 선택을 다시 할 수 있습니다 — 파일을 다시 고를 필요 없습니다.

### 건축물대장 (브리지)
- **SHP 모드**: shp·dbf·prj를 함께 선택(또는 ZIP). 좌표계 자동 판별, 선으로 그려진 부지도 면 처리.
- **주소 모드**: 한 줄에 주소 하나(지번·도로명 모두). Excel A열 불러오기 지원.
- vworld 면적 제한(10km²) 초과 부지는 자동으로 타일 분할 조회됩니다.

### PDF 표 → 엑셀 (브리지)
- [1. 표 찾기] → 찾은 표 목록에서 **저장할 표 선택** → [2. 엑셀로 저장].
- 연속 페이지에 걸친 표는 하나로 병합되고 중간 반복 머리행은 제거됩니다.
- 페이지 번호는 **PDF 물리 쪽번호**(뷰어에 보이는 순번)입니다.
- ⚠ 스캔 이미지 PDF는 표 추출 불가 — 텍스트 레이어가 있어야 합니다(OCR은 문서→MD 경유).

### 문서 → MD
- **웹(기본)**: PDF·DOCX·XLSX를 끌어다 놓고 변환. EIA 번호체계 헤딩 승격·표 보존.
- **브리지**: HWP·HWPX·스캔 PDF(OCR)·폴더 일괄 → 결과는 그 폴더의 `markdown_output/`.

### EIASS 수집
- **FILE_SEQ 직접(웹)**: 번호 붙여넣기 → 전체 다운로드.
- **사업코드 자동(브리지)**: 코드 입력 → [목록 조회] → 절차·장코드·파일명에서 체크 선택 → 다운로드.
  「ZIP으로 묶기」 체크 시 `{코드}.zip` 동시 생성.

### 쪽번호 (브리지)
- [폴더 선택] → 조건 지정 → 스캔 → 확인 후 적용. **폴더 안 전체 .hwp를 일괄 처리**합니다.
- **빈 페이지를 만들지 않고 쪽번호로만 양면 정합**을 맞춥니다(문서 구조 불변).
- 간지 1장/2장, A3 뒷면 결번/물리공백을 **사용자가 선택**합니다 — 같은 인쇄 결과를 내는
  두 작성 방식이라 도구가 추정하지 않습니다.
- 스캔 결과가 선택 조건과 다르면 비고에 빨간 경고가 뜹니다(그대로 진행하면 쪽번호가 밀립니다).

### HWP → PDF (브리지)
- [폴더 선택] → 실행. 모아찍기 무시·단면 출력, 이미지화 없이 텍스트 보존.

## 5. 문제 해결

| 증상 | 조치 |
|---|---|
| **고쳤다는데 화면이 그대로** | `Ctrl+Shift+R` (하드 리프레시). 왼쪽 위 웹 버전 배지로 확인 |
| 웹과 브리지 **버전 숫자가 다름** | **정상입니다.** 별개 프로그램입니다. 실제 불일치는 상태칩에 ⚠로 표시됩니다 |
| 브리지가 여러 개 떠 있음 | v3.25.0부터 중복 실행이 차단됩니다. 예전 버전으로 쌓인 것은 웹을 닫으면 정리됩니다 |
| 브리지 칩이 계속 ○ 미연결 | bat 창이 떠 있는지 → 15초 대기 → 상태칩을 눌러 안내 확인 |
| 칩에 **⚠ 갱신 필요** | 열려 있는 브리지를 닫고 `run_bridge.bat` 다시 실행 |
| "토큰 불일치" | 브리지 콘솔의 토큰을 다시 복사해 ⚙ 설정에 입력 |
| "승인된 경로가 아닙니다" | [폴더 선택] 버튼으로 다시 지정 (보안상 승인된 폴더만 접근) |
| 지오코딩이 전부 실패 | **입력 좌표계**를 확인하세요. 매핑 화면의 권장 안내를 따르면 됩니다 |
| 지도에 위성영상이 없음 | ⚙ 설정에 vworld 키 입력 (없으면 OpenStreetMap으로 동작) |
| vworld 오류 | 키 확인 · 만료 여부(vworld 포털) 확인 |

## 6. 구조 (관리자용)

```
unified_workbench/
├─ index.html            셸 (탭·상태칩·설정)
├─ shared/               tokens.css·ui.css·app.js·bridge.js·keys.js
│                        geo.js·mapview.js(지도 공통)·geocode.js(vworld 래퍼)
├─ modules/              parcel·md·eiass·hwp·pdf2excel·photo·geocode
├─ vendor/               shpjs·proj4·xlsx·pdf.js·mammoth·leaflet·Pretendard (전부 동봉)
├─ bridge/               bridge_server.py + photo_exif.py + hwp_pagenum.py + run_bridge.bat
├─ tests/                Node 회귀 테스트 (탭·모듈·좌표계·열매핑·브리지호환)
└─ poc/                  브리지 통신 진단 페이지
```

### 배포

- **웹**: main 푸시 → GitHub Pages 자동 반영(1~2분).
  ⚠ `shared/app.js`의 `V`와 `index.html`의 `?v=`를 **함께 올려야** 브라우저가 새 파일을 받습니다.
  모듈 간 import에도 `?v=`가 붙어 있어야 합니다 — 빠뜨리면 그 파일만 캐시에 고착됩니다.
- **브리지**: `bridge/build_bridge.bat`로 exe 빌드(Windows 전용).
  빌드 성공은 합격이 아닙니다 — **`bridge/verify_bridge.bat` 더블클릭**으로 저장소 실행본과
  기능 목록을 대조하세요(`[OK] Verified`가 나와야 배포 가능).

  ⚠ **`[BLOCKED]`가 뜨면 Windows가 exe 실행 자체를 막은 것입니다**(스마트 앱 제어·WDAC·AppLocker).
  빌드 결함이 아니라 그 PC에서 서명 없는 프로그램 실행이 금지된 상태입니다.
  **빌드 직후에만 막히는 경우가 많습니다** — Defender/SmartScreen이 처음 보는 파일을
  클라우드 평판 조회 동안 잡아둡니다. 몇 분 뒤 `verify_bundle.py`만 다시 돌리면 됩니다
  (재빌드 불필요). 파일이 바뀌면 평판이 새로 매겨지므로 빌드할 때마다 반복될 수 있습니다.
  **이 PC에서 막히면 받는 PC에서도 막힐 가능성이 높으므로**, exe 배포 대신
  Python + `run_bridge.bat` 방식을 검토하세요(파이썬 3.10+만 있으면 됩니다).
  `BRIDGE_VERSION`은 **브리지 코드를 실제로 고칠 때만** 올립니다(웹 버전에 맞춰 올리면
  바뀐 것도 없는데 사용자에게 재시작을 요구하게 됩니다).

### 참조 구조

브리지는 기존 도구(`convert_to_md`·`EIASS`·`hwp2pdf`·`pdf2excel`)를 **import로 참조**합니다 —
코드 복제가 없으므로 원본을 개선하면 자동 반영됩니다. 경로는 하드코딩하지 않고
`claude_paths.resolve()`로 이름을 통해 찾습니다(폴더가 재편돼도 인덱스만 다시 만들면 됩니다).

API 키는 코드에 두지 않습니다 — 웹은 브라우저 localStorage, 부서 스크립트는
루트 `api_keys.py`가 `resources_index.json`·환경변수에서 조달합니다.
