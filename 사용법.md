# EIA Workbench 사용법

> 웹 UI: **https://jingeun-git.github.io/eia-workbench/** — 설치 없이 접속만 하면 됩니다.
> 이 문서는 최초 1회 설정과 도구별 사용법을 담습니다.

## 0. 한눈에 보기

| 도구 | 브리지 필요? | 하는 일 |
|---|---|---|
| 건축물대장 | ❌ | 사업지구 SHP/주소 목록 → 필지별 건축물대장 → Excel |
| md 변환 | ❌ (HWP·OCR만 ⭕) | PDF·DOCX·XLSX → 마크다운 (브라우저 안 처리) |
| EIASS | ❌ (사업코드 자동은 ⭕) | FILE_SEQ 직접 또는 사업코드 자동탐색 다운로드 |
| HWP→PDF · 쪽번호 | ⭕ | 폴더 단위 한컴 일괄 작업 |

**브리지** = PC에서 실행하는 보조 프로그램. 브라우저가 못 하는 일(한컴 조종·HWP 변환·EIASS 검색 API)을 대신합니다.
꺼져 있어도 ❌ 표시 도구는 전부 정상 동작합니다.

## 1. 최초 1회 — API 키 설정 (건축물대장용)

1. 웹 UI 접속 → 우상단 **⚙ 설정**
2. **vworld API 키**: [vworld.kr](https://www.vworld.kr) 회원가입 → 인증키 발급 → 입력
3. **공공데이터포털 키**: [data.go.kr](https://www.data.go.kr) → "건축HUB 건축물대장 정보 서비스" 활용신청
   → 마이페이지의 **일반 인증키(Decoding)** 입력
4. 저장 — 키는 이 브라우저의 localStorage에만 남고 어디에도 전송되지 않습니다

> 기존 「편입지적건축물대장조회.exe」에서 쓰던 키 그대로입니다.

## 2. 브리지 실행 (한컴·한글문서 변환·EIASS 자동탐색 쓸 때만)

**`D:\Claude\99.Tools\unified_workbench\bridge\run_bridge.bat` 더블클릭 — 끝.**

브라우저가 자동으로 열리면서 토큰이 스스로 등록되고, 상태칩이 **● 연결됨**으로 바뀝니다.
토큰 복사·붙여넣기는 필요 없습니다.

- 요구사항: Windows Python **3.10+** (한컴 작업은 한컴오피스 + pywin32 추가)
- 창을 닫으면 브리지 기능만 꺼지고 웹 3종은 계속 동작합니다
- 자동으로 안 열리면: 콘솔에 표시된 토큰을 웹 UI ⚙ 설정에 수동 입력
- 토큰은 `bridge/bridge_config.json`에 저장 — **이 파일은 공유·커밋 금지**

## 3. 도구별 사용법

### 건축물대장
- **SHP 모드**: shp·dbf·prj를 함께 선택(또는 ZIP). 좌표계는 자동 판별(중부/동부/서부/구Bessel),
  경계가 선(LineString)으로 그려진 부지도 자동으로 면 처리됩니다.
- **주소 모드**: 한 줄에 주소 하나(지번·도로명 모두 가능). Excel A열 불러오기 지원.
- 결과 확인 후 [Excel 저장] — 기존 exe와 동일 서식(남색 헤더·줄무늬).
- vworld 면적 제한(10km²) 초과 부지는 자동으로 타일 분할 조회됩니다.

### md 변환
- **웹(기본)**: PDF·DOCX·XLSX를 끌어다 놓고 변환. EIA 번호체계 헤딩 승격·표 보존·UTF-8 BOM 저장.
- **브리지 경로**: HWP·HWPX·스캔 PDF(OCR)·폴더 일괄 → [폴더 선택] 후 실행,
  결과는 그 폴더의 `markdown_output/`에 저장.

### EIASS
- **FILE_SEQ 직접(웹)**: 번호 붙여넣기 → 전체 다운로드. 브라우저 기본 다운로드 폴더로 저장(1.2초 간격).
- **사업코드 자동(브리지)**: 코드 입력 → [목록 조회] → 절차(초안/본안/보완…)·장코드·파일명 목록에서
  **체크 선택** → 저장 폴더 선택 → [선택 다운로드]. 「ZIP으로 묶기」 체크 시 `{코드}.zip` 동시 생성.
  유형은 자동 판별되며 소규모/평가/전략/사후 강제 지정 가능(전략은 평가와 같은 경로로 조회).

### HWP→PDF / 쪽번호 (브리지)
- 공통: [폴더 선택] → 실행. **폴더 안 전체 .hwp를 일괄 처리**합니다.
- 쪽번호: 시작 번호 입력 → 연속 재부여 + 파일명 쪽 표기 갱신.
  (장 짝수끝·홀수시작, A3 홀수 부여·공란 숨김 등 제어 기능은 SYS-30으로 개발 예정)

## 4. 문제 해결

| 증상 | 조치 |
|---|---|
| 칩에 **⚠ 진단 스텁 감지** | 예전 PoC 스텁 창(bridge_stub.py)이 켜져 있음 — 그 창을 닫고 run_bridge.bat 실행 |
| 브리지 칩이 계속 ○ 미연결 | bat 창이 떠 있는지 → 15초 대기 → 그래도 안 되면 bat 창의 메시지 확인 |
| "토큰 불일치" | 브리지 콘솔의 토큰을 다시 복사해 설정에 입력 |
| 화면이 예전 그대로 | `Ctrl+Shift+R` (강제 새로고침) |
| 건축물대장 "승인된 경로가 아닙니다" | [폴더 선택] 버튼으로 다시 지정 (보안상 승인된 폴더만 접근) |
| vworld 오류 | ⚙ 설정의 키 확인 · 키 만료 여부(vworld 포털) 확인 |

## 5. 구조 (관리자용)

```
unified_workbench/
├─ index.html            셸 (탭·상태칩·설정)
├─ shared/               tokens.css·ui.css·app.js·bridge.js·geo.js
├─ modules/              parcel·md·eiass·hwp (도구별 모듈)
├─ vendor/               shpjs·proj4·xlsx·pdf.js·mammoth·Pretendard (전부 동봉)
├─ bridge/               bridge_server.py + run_bridge.bat
└─ poc/                  브리지 통신 진단 페이지
```

- 배포: main 푸시 → GitHub Pages 자동 반영 (1~2분)
- 브리지는 기존 도구(convert_to_md·EIASS·hwp2pdf·배포용)를 **import/서브프로세스로 참조** — 코드 복제 없음.
  따라서 기존 도구를 업데이트하면 브리지 경로에 자동 반영됩니다.
